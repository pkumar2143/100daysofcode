# Modeling real-world objects.

Restaurant example, instead of running each task (waiting, cooking, cleaning, etc) by yourself, you hire folks, 
and each has their own attributes and actions (methods)

Waiter
	+ Has: holding_plate, tables_responsible
	+ Does: take_order, take_payment

We could have multiple waiters (objects) based on the same blueprint (class)

waiter = WaiterBlueprint()
  ^               ^
  |				  |
  object        class

waiter.holding_plate  #attribute
waiter.take_payment() #method
